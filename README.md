# VibeYeeter3000

An internal PaaS for AI-assisted application development by non-technical users.

VibeYeeter3000 lets anyone at the company build and deploy applications using AI coding agents (Claude, Cursor, etc.) without needing to understand Kubernetes, Terraform, or CI/CD pipelines. The platform provisions repos, manages infrastructure, runs deployments, and provides a dashboard where non-coders can see what's running, roll back, inspect migrations, and manage secrets.

Engineers vibe-code in their repos. The platform does the rest.

## What it is

- **For non-technical users**: a dashboard to see deployments, roll back, view migration history, manage secrets, and approve infrastructure changes — no terminal required
- **For AI coding agents**: a standardized repo structure, CLAUDE.md conventions, and automated pipelines so agents know exactly how to build, test, and ship
- **For the platform team**: a GitHub App + Terraform runner + Kubernetes operator that governs every app's lifecycle

## Core stack

| Layer | Technology |
|---|---|
| App language | TypeScript |
| App framework | Next.js 14+ (App Router) |
| ORM | Drizzle |
| Database | PostgreSQL (CloudNative PG) |
| Container runtime | Docker |
| Orchestration | AWS EKS (Kubernetes) |
| Infra-as-code | Terraform |
| CI/CD | GitHub Actions (generated by platform) |
| Image registry | AWS ECR |
| Secrets | AWS Secrets Manager + External Secrets Operator |
| Ingress auth | Cloudflare Access → JumpCloud SSO |
| Platform auth | JumpCloud SAML |
| GitHub integration | GitHub App (`vibeyeeter-bot`) |

## Repo structure

```
vibeyeeter3000/
├── apps/
│   ├── web/          Next.js control plane UI
│   └── api/          Fastify API server
├── packages/
│   ├── types/        Shared TypeScript types
│   └── github-app/   GitHub App webhook handling + repo operations
├── services/
│   └── tf-runner/    Terraform runner service
├── infra/
│   ├── platform/     Platform's own AWS infra (EKS, ECR, S3, DynamoDB)
│   └── modules/      Reusable Terraform modules for app infra
├── k8s/
│   ├── platform/     Platform's own Kubernetes manifests
│   └── app-chart/    Helm chart for managed apps
├── scripts/
│   └── bootstrap.sh  One-time platform setup
└── docs/
```
